# DIMENSION FILE SCHEMA
# Every dimension file must complete every field.
# Incomplete fields fail QA regardless of content quality elsewhere.

---

## FILE ID
[exact match to master list — character-for-character]

## DIMENSION TYPE
[one of: Moral Register / Pacing / Supernatural Mechanic / Plot Engine / Protagonist Structure / Tone Register]

## ONE-LINE DEFINITION
[One sentence. Pulled verbatim into the Master Index. Must be functional, not descriptive.]

## FUNCTIONAL DEFINITION
[3-5 paragraphs. What does this dimension setting actually do to a story when active? Not what it is — what changes in narrative, prose, and reader experience when it is present. Ground this in craft, not theme.]

## READER PSYCHOLOGY
[2-3 paragraphs. Why does a reader respond to this setting? What does it give them that adjacent settings do not? What does it withhold? Be specific to this setting.]

## PROSE SIGNATURE
[4-6 specific prose behaviors. Each must name a concrete craft decision, not a general quality. Format: behavior — what it does in practice. "Write with restraint" fails. "Scene endings resist emotional resolution — end on information or action, not feeling" passes.]

## CALIBRATION EXAMPLES
[3-4 examples from published works. Each must name a specific scene or passage and describe what the prose is doing at the sentence level. Plus exactly one NEAR-MISS example with the failure named in one specific sentence identifying a craft decision, not a quality judgment.]

## FAILURE MODES
[3-5 specific failure patterns. Each must name a concrete craft failure, not a vague quality problem. Include at least one published work that demonstrates the failure.]

## AI WRITING DEFAULTS TO AVOID
[3-5 entries. Exact format required:
[Specific default pattern the writing system will produce] → [Why this pattern fails this specific dimension setting] → [Specific alternative behavior]
Every entry must be specific to this dimension setting. If the entry could appear in any other dimension file, it fails.]

## CONFLICT WITH OTHER DIMENSIONS
[Every conflict. Format per entry:
- Conflicts with: [file-id]
- What creates the friction: [specific mechanism]
- Default winner: [file-id] — [one sentence why]
- Override condition: [specific circumstance under which the non-default dimension takes precedence]
No conflict listed without an override condition. No conflict omitted because it is complicated.]

## COMBINATION NOTES
[Every clean combination. Format per entry:
- Combines with: [file-id]
- Emergent effect: [what this combination produces that neither dimension produces independently — one specific sentence]
"Works well with X" is not sufficient.]
