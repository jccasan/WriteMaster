Build a production-ready MVP called STORY FORGE.

STORY FORGE is a fiction-only AI editing studio for full manuscripts. It must support separate editorial services, shown in the normal professional order:

1. Editorial Assessment
2. Developmental Editing
3. Copy Editing
4. Proofreading
5. Fact-Checking

It must also include:
- AI Beta Reader Suite
- Story Structure Analyzer
- Character Consistency Tracker
- Scene Purpose Scanner

IMPORTANT BUILD RULES
- Do not give mockups only.
- Do not stop at scaffolding.
- Build a working MVP end to end.
- Include seeded sample data and a sample manuscript project.
- Include editable prompts in config files.
- Include a real chunk-processing and synthesis flow.
- Prioritize working functionality over polish, then refine the UI.

STACK
Use this unless there is a strong reason not to:
- Next.js 14+ with TypeScript
- Tailwind CSS
- shadcn/ui
- Prisma
- SQLite for MVP
- Server actions or API routes
- Simple background job runner in app for MVP
- DOCX/TXT/PDF text extraction libraries
- PDF and DOCX export support
- Modular LLM provider abstraction
- Modular web research service abstraction for fact checking

If a simpler full-stack TypeScript architecture is more stable on Replit, use it. Prioritize reliability.

CORE PRODUCT BEHAVIOR
The app must:
- ingest full manuscripts
- parse and detect chapters
- chunk the manuscript into 3 to 5 chapter groups
- analyze each chunk independently
- create structured internal memory as it goes
- synthesize final manuscript-level outputs
- deduplicate repeated findings
- remove false positives that are resolved later in the manuscript
- track recurring unresolved issues
- support multiple revision versions
- compare editorial results between revisions

SUPPORTED INPUTS
Manuscript input:
- .docx
- .txt
- .pdf if extractable
- pasted text

Support files:
- outline

REFERENCE PRIORITY
When support files exist, use them actively in analysis, in this priority order:
1. story bible / outline / canon / continuity files
2. uploaded editorial frameworks
3. system-extracted outline
4. manuscript evidence itself

EDITORIAL PHILOSOPHY
The analysis framework should reflect strong editorial craft such as:
- reader contract through empathy and curiosity
- suspense through unresolved tension
- conflict and forward motion in openings
- causality over convenience
- continuity and time discipline
- scene integrity
- scenes doing more than one job
- character movement at scene level
- theme and anti-theme pressure
- character lie versus truth
- want versus need
- structural turning points
- pacing and proportion
- endings with payoff and aftermath

Do not force rigid formulas where the manuscript is intentionally doing something else successfully.

MVP USER FLOW
1. User creates project
2. User uploads manuscript
3. User uploads optional support files
4. App parses files
5. App detects chapters
6. App creates revision version
7. User chooses analysis type
8. App chunks manuscript into 3 to 5 chapter groups
9. App runs chunk analyses
10. App updates structured memory after each chunk
11. App runs synthesis
12. App renders final report
13. User browses report, issues, structure, characters, scenes, beta reader output, fact checks
14. User uploads revised version later and compares

- Chapter One

If imperfect, expose a basic manual override UI later, but at minimum store detected boundaries cleanly.

CHUNKING RULES
- chunk by 3 to 5 chapters
- default to 4 chapters
- preserve chapter order
- if chapters are extremely uneven, prefer chapter grouping over raw token splitting unless a chapter is too large to process
- if no chapters are found, use balanced segments and label them segments

SYNTHESIS ENGINE
This is mandatory and central.

Final synthesis must:
- merge duplicate issues
- merge semantically similar issues
- remove or downgrade issues clearly resolved later
- distinguish temporary uncertainty from true unresolved problems
- preserve recurring and systemic problems
- prioritize final findings by editorial importance

Issue states:
- introduced
- active
- resolved_later
- reintroduced
- uncertain
- dismissed
- merged

SYNTHESIS LOGIC EXAMPLES
- If a plot thread looks unresolved in chunk 2 but resolves in chunk 8, do not present it as unresolved in the final report.
- If an injury is forgotten for many chapters and later reappears with no explanation, keep it as an active continuity issue.
- If early passivity is part of a clear arc, do not mislabel it as a character failure unless the transition is mishandled.

EDITOR MODULE SPECS

1. EDITORIAL ASSESSMENT
Role:
Senior overview editor.

Output:
A polished editorial letter.

Focus:
- premise
- hook
- genre fit
- macro structure
- marketable strengths
- biggest weaknesses
- emotional engagement
- pacing at manuscript level
- top revision priorities

Sections:
- Overview
- What the manuscript is doing
- What is working
- Main concerns
- Recommended next steps
- Best next editorial pass

2. DEVELOPMENTAL EDITING
Role:
Story architect.

Output:
- editorial letter
- chapter findings
- issue list
- targeted scene/chapter notes
- rewrite direction notes

Focus:
- plot and causality
- suspense
- arc logic
- pacing
- scene purpose
- exposition
- subplot integration
- world-rule consistency
- payoff and setup
- ending performance

3. COPY EDITING
Role:
Voice-preserving line editor.

Focus:
- clarity
- repetition
- awkward syntax
- dialogue clarity
- tone consistency
- terminology consistency
- overwritten or flat prose
- cliché patterns
- filler
- paragraph rhythm

Preserve voice when it works.

4. PROOFREADING
Role:
Mechanical cleanup pass.

Focus:
- spelling
- grammar
- punctuation
- capitalization
- formatting
- quotation consistency
- duplicate words
- missing words

5. FACT-CHECKING
Role:
Canon and realism verifier.

Internal:
- timeline
- injuries
- continuity
- knowledge state
- object continuity
- location continuity
- world rules
- canon adherence

External:
- historical facts
- geography
- travel plausibility
- legal and institutional realism
- military / intelligence / police realism
- medicine
- weapons
- science
- technology

Use a modular research provider for external checks and store citations/source references when used.
